執行步驟

資料來源已確認，下一步是匯入 Google Sheet 並部署 Apps Script API。
1

建立資料快照

從蝦皮獲利表、每日入庫清單、AjinERP 庫存盤點表輸出標準 CSV，先核對筆數與欄位再匯入。

2

匯入 Google Sheet 資料庫

使用 products、inventory、expiry_lots、daily_checks 四張核心表；效期批號先保留人工補登欄位。

3

部署 Apps Script API

提供首頁資料、商品查詢、今日盤點、手動同步與盤點回寫，讓前端可以讀寫 Google Sheet。

4

改接正式資料

把目前畫面上的示範資料改為 API 回傳資料；若尚未部署網址，畫面會保留示範資料作為備援。

5

手機與門市測試

確認每天 2 個類別、約 10 個 SKU 的盤點清單正確產生，盤點回寫後能更新庫存與最後盤點日。

資料表規劃

第一版必要分頁。
產品主檔
產品貨號、商品名稱、A-J系列分類、成本、售價、商品大類、盤點優先級、盤點頻率、安全庫存、是否啟用盤點。
庫存現況 / 效期批次
產品貨號、現有庫存、最近效期、批次效期、批次數量、儲位、最後盤點日、效期狀態。
每日盤點任務 / 盤點紀錄
日期、盤點類別、產品貨號、安排原因、實際庫存、效期、狀態、備註、填寫人、送出時間。
目前資料來源
商品主檔：蝦皮獲利計算表_含稅版_updated.xlsx；庫存同步：產品進貨單 / 庫存盤點表；每日入庫：產品進貨單底下各供應商入庫清單。效期批號需另由進貨單或人工效期表補齊。
